📋 TEORETICKÝ PRÍSTUP - ARCHITEKTÚRA SYSTÉMU
1. VRSTVOVÁ ARCHITEKTÚRA (Separation of Concerns)
Prečo: Každá vrstva má svoju zodpovednosť, ľahko sa testuje a rozširuje.
Vrstva 1: Configuration Layer

Účel: Centralizovaná konfigurácia (API keys, modely, parametre)
Triedy:

AISettings - obsahuje všetky nastavenia z .env súboru
get_ai_settings() - singleton pattern pre prístup ku konfigurácii


Prečo singleton: Zabezpečíš že konfigurácia sa načíta len raz

Vrstva 2: Data Layer (Schemas)

Účel: Definícia dátových štruktúr, validácia
Triedy:

ExtractionRequest - čo prichádza od klienta
ExtractionResponse - čo vraciaš klientovi
ExtractedData - štruktúra extrahovaných dát
DodavatelInfo, Polozka, atď. - vnorené objekty


Prečo Pydantic: Automatická validácia, JSON serialization, type hints

Vrstva 3: Client Layer

Účel: Komunikácia s externými API (OpenAI)
Triedy:

OpenAIClient - wrapper okolo OpenAI API
Metódy: extract_structured_data(), _build_prompt()


Funkcie:

Retry logic - ak API zlyhá, skús 3x s exponenciálnym čakaním
Prompt engineering - few-shot learning s príkladmi
Error handling - čo keď API vráti nevalidný JSON?



Vrstva 4: Service Layer

Účel: Business logika, orchestrácia
Triedy:

AIExtractionService - hlavná služba
Metódy: extract(), _calculate_cost(), _log_metrics()


Zodpovednosti:

Volá OpenAIClient
Počíta metriky (čas, tokeny, cena)
Loguje všetko do súborov
Validuje výstupy



Vrstva 5: API Layer

Účel: REST endpoints pre klienta
Endpoints:

POST /api/v1/extraction/text - pošli text, dostaň JSON
POST /api/v1/extraction/file - upload súboru


FastAPI features:

Automatická validácia cez Pydantic
Swagger dokumentácia zadarmo
Async/await pre performance




🧪 TESTOVANIE A EVALUÁCIA
2. GROUND TRUTH DATASET
Účel: Máš referenčné dáta na porovnanie
Štruktúra:

Testovacie dokumenty - 20-50 reálnych príkladov
Expected outputs - manuálne overené správne odpovede
Metadata - obtiažnosť (easy/medium/hard), typ dokumentu

Kategórie testov:

Jednoduché - základná faktúra s pár poliami
Stredné - komplexnejšie dokumenty, viac položiek
Ťažké - neúplné dáta, neštandardný formát
Edge cases - chýbajúce polia, divné formátovanie


3. RAGAS FRAMEWORK (Pre bakalárku perfektné!)
Čo je RAGAS: Framework na evaluáciu AI systémov, hlavne RAG (Retrieval Augmented Generation)
Metriky ktoré použiješ:
Faithfulness (Vernosť)

Meria: Nekonfabuluje AI? Extrahuje len to čo je v dokumente?
Skóre: 0.0 - 1.0 (1.0 = perfektné)
Pre bakalárku: Dokážeš že AI nevymýšľa dáta

Answer Relevancy (Relevantnosť)

Meria: Je odpoveď relevantná k zadaniu?
Dôležité pre: Či AI extrahuje správne veci

Answer Correctness (Správnosť)

Meria: Zhoda s ground truth
Porovnáva: Tvoj expected output vs AI output
Key metrika pre bakalárku

Context Precision (Presnosť kontextu)

Meria: Používa AI len relevantné časti dokumentu?

Ako RAGAS funguje:

Máš dokument (context)
Máš question ("extrahuj faktúru")
Máš AI answer (extrahované JSON)
Máš ground truth (správna odpoveď)
RAGAS automaticky vypočíta skóre


4. CUSTOM METRIKY (Pre bakalárku bonus body)
Field-level accuracy:

Koľko % polí je správne?
Napr. datum správne v 95% prípadov, ale IČO len v 80%

Token efficiency:

Koľko tokenov spotrebuje jeden dokument?
Sleduj cost over time

Response time:

Priemerný čas odpovede
95th percentile (väčšina requestov rýchlejšia než X sekúnd)

Error categories:

Parsing errors (nevalidný JSON)
Missing fields (chýbajúce polia)
Wrong format (zlý dátumový formát)
Hallucinations (vymyslené dáta)


📊 EXPERIMENTS A POROVNANIA
5. ITERATÍVNE ZLEPŠOVANIE (Pre bakalárku - ukáž postup)
Experiment 1: Baseline (Základ)

Jednoduchý prompt bez príkladov
Len "extrahuj dáta z tohto dokumentu"
Očakávaný výsledek: ~60-70% presnosť

Experiment 2: Few-shot Learning

Pridáš 2-3 príklady do promptu
Ukážeš AI ako má výstup vyzerať
Očakávaný výsledek: ~80-85% presnosť
Pre bakalárku: Grafy porovnania baseline vs few-shot

Experiment 3: Chain-of-Thought

Povieš AI aby "rozmýšľalo nahlas"
"Najprv nájdi číslo faktúry, potom dátum..."
Očakávaný výsledok: ~85-90% presnosť
Trade-off: Viac tokenov = vyššia cena

Experiment 4: Fine-tuning modelu

Natrénuješ vlastný model na tvojich dátach
Náročné ale efektné pre bakalárku
Porovnáš cost: GPT-4 vs fine-tuned GPT-3.5
